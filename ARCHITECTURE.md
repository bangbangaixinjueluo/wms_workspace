# ARCHITECTURE

## 本文用途
用于描述系统整体架构、核心模块关系、关键数据流与依赖边界，帮助开发者快速建立全局理解。

## 适用范围
适用于整个系统的服务划分、前后端交互、数据存储、异步任务与外部集成。

## 如何维护
- 发生架构级变更时同步更新。
- 新增核心服务、消息链路或基础设施依赖时补充本文。
- 模块级细节放在各自设计文档中，本文只保留系统级视角。

## 系统概览
- 当前为 WMS 相关工作区，预计包含前端界面、业务服务、数据库与运维支撑能力。
- 架构文档默认从“用户动作 -> 应用服务 -> 持久化/外部系统”这一主链路组织。
- 未确认的实现细节暂用占位描述，不伪造现状。

## 关键数据流
### 典型业务链路
1. 用户在前端发起业务操作。
2. 应用层校验参数与权限。
3. 领域服务执行业务规则并写入数据库。
4. 如有需要，触发异步任务、消息或外部系统同步。
5. 前端读取最新状态并反馈用户。

## 依赖边界
- 前端不直接依赖数据库。
- 领域逻辑不应泄漏到展示层。
- 外部系统集成应集中在适配层或集成层，避免散落在业务代码中。
