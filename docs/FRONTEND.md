# FRONTEND

## 本文用途
用于沉淀前端层面的通用约定，包括页面组织、组件边界、状态管理、接口交互、可访问性与交付质量要求。

## 如何维护
- 当前端框架约定、目录组织或 UI 规范变化时更新。
- 遇到重复出现的实现偏差，应沉淀为本文规则。

## 页面与组件边界
- 页面负责路由级编排与数据装配。
- 业务组件负责具体领域交互，不承载全局副作用。
- 基础组件应尽量无业务语义，可复用、可组合。

## 状态与数据获取
- 本地状态仅保存当前页面直接需要的 UI 状态。
- 服务端数据应通过统一请求层获取。
- 长生命周期状态需明确来源、更新时机与失效策略。

## 体验要求
- 表单反馈及时明确。
- 异常状态、空状态、加载状态必须可见。
- 关键操作需要具备防误触与结果确认机制。

## 质量要求
- 新页面至少覆盖主流程交互验证。
- 复杂组件应保留使用说明或示例。
- 样式与交互改动应关注移动端与桌面端兼容性。
