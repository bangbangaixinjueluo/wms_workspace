# Core Beliefs

## 本文用途
用于记录系统设计中的长期信念与默认原则，帮助后续设计在一致的价值观下演进。

## 如何维护
- 只记录足够稳定、可长期复用的原则。
- 如果原则失效，应说明触发变化的背景。

## 核心信念
- 优先清晰性，再追求技巧性实现。
- 优先领域边界清楚，再追求抽象复用。
- 优先可恢复与可排障，再追求路径最短。

## 设计取向
- 数据结构应服务于业务语义，而不是仅服务于技术便利。
- 关键流程必须能观察、能解释、能追溯。
- 复杂度要集中在少数明确位置，不要全局扩散。
