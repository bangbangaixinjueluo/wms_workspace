# Clarification Rules

## 什么时候要提澄清问题

只有当答案会明显改变方案时才提问。

## 优先澄清的问题类型

1. 生效范围  
   只对哪类单据、哪类业务类型、哪类角色生效

2. 终态定义  
   “完结”“完成”“关闭”“终结”在业务上是不是同一个意思

3. 反操作  
   是否允许反完结、反关闭、重新打开

4. 前端规则还是后端规则  
   是只控制显示，还是后端也要严格校验

5. 高风险链路  
   是否涉及钱包、银行卡、委外、维修、退厂、共享库存

## 不要问的低价值问题

- 用户已经明确给出的内容
- 不影响方案选型的细枝末节
- 代码里可以直接查到的东西
