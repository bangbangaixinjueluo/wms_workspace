# Output Contract

## 标准输出结构

### 需求理解
- 把业务话术翻译成系统里的真实对象
- 明确这是页面规则、接口规则、状态机规则，还是口径规则

### 涉及模块
- 主模块
- 上下游模块
- 使用到的总表

### 影响范围
- 页面或按钮
- 接口或服务入口
- 状态流
- 字段与累计值
- 跨模块回写

### 方案设计
- 前端层怎么改
- 后端层怎么改
- 权限建议
- 数据处理建议
- 日志/消息建议

### 风险点
- 聚合状态
- 累计字段
- 钱包/银行卡
- 来源挂接
- 历史数据兼容

### 待确认问题
- 面向客户的澄清问题
- 面向开发的代码确认点

## 输出语气要求

- 结论先行
- 风险明确
- 不空泛
- 不默认进入编码
