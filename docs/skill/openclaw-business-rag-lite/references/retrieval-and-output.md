# Retrieval And Output

## 1. 检索流程

对“业务需求评估”类问题，推荐固定流程：

1. 识别需求关键词
2. 先召回总表层
3. 再召回模块正文层
4. 最后召回图谱层
5. 做一次去重和重排
6. 输出结构化评估

## 2. 重排规则

优先级建议：
1. 总表层命中
2. 目标模块正文
3. 上下游模块正文
4. 图谱层

不要只按相似度排序。
对这套业务文档，很多高价值信息在总表层，不在模块正文第一段。

## 3. 问题到文档的优先映射

### 状态类问题
优先读：
- `state-matrix.md`
- 对应模块正文
- 对应状态图

### 字段口径类问题
优先读：
- `business-metrics.md`
- `field-semantics.md`
- 对应模块正文

### 联动与影响范围
优先读：
- `cross-module-writeback.md`
- `change-impact.md`
- 对应模块正文

### 外部系统与共享库存
优先读：
- `external-interactions.md`
- `sharestock.md`
- `shareparty.md`
- `synctask.md`
- `deliverybill.md`

## 4. 标准输出模板

默认输出这 6 段：

### 需求理解
- 用业务语言复述需求
- 翻译成系统真实模块、状态、字段

### 涉及模块
- 主模块
- 上下游模块
- 相关总表

### 影响范围
- 页面/按钮
- 接口
- 状态机
- 字段
- 回写链

### 方案设计
- 前端改动点
- 后端改动点
- 权限点
- 数据与状态处理建议

### 风险点
- 聚合状态风险
- 累计字段风险
- 钱包/银行卡风险
- 历史数据兼容风险

### 待确认问题
- 对客户/产品需要确认的口径
- 文档已覆盖但代码仍需确认的点

## 5. 输出边界

默认不要直接：
- 写业务代码
- 给出数据库迁移脚本
- 假设某个状态一定来自前端按钮

默认应该：
- 先做评估
- 明确假设
- 明确风险
- 明确哪些点需要回源码确认
