# 技术债追踪表

## 本文用途
用于长期跟踪跨模块、跨迭代的技术债问题，支持按优先级、影响面与处理状态统一管理。

## 如何维护
- 新发现的技术债要尽快登记。
- 同一问题被多个任务反复提及时，应合并为单一条目。
- 已处理项可保留历史记录，不直接删除。

## 状态定义
- `open`：已确认，尚未开始处理。
- `planned`：已进入计划。
- `in-progress`：处理中。
- `blocked`：存在阻塞。
- `resolved`：已解决，待观察或已验证。

## 优先级建议
- `P0`：影响生产稳定性或安全，需立即处理。
- `P1`：影响核心链路，应尽快排期。
- `P2`：影响效率或维护成本，纳入近期计划。
- `P3`：优化类问题，按资源情况处理。
