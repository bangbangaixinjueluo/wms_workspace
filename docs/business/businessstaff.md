# 业务员

## 1. 模块定位

- 对应表：`wh_business_staff`
- 入口：`BusinessStaffController`、`BusinessStaffServiceImpl`
- 角色：客户、订单、分区经理链和移动端业务员身份识别宿主

## 2. 核心职责

- 维护业务员与系统用户、部门、业务分区的绑定
- 支持业务员分页、导出、名称模糊查找
- 提供按用户 ID 取业务员、按分区取业务员、按名称映射
- 为 App 端返回业务员身份与岗位信息

## 3. 代码结论

- `id` 直接等于 `userId`，业务员创建时即固定
- 创建时会校验系统用户存在、用户所属部门存在
- 若查到同 `userId` 的已删除记录，会先取消删除再更新
- `getStaffInfo(...)` 会根据“是否是分区经理 / 是否是普通业务员”判断岗位

## 4. 关注点

- `⚠️ deleteBusinessStaff(...)` 是物理删除，未见对客户、订单、统计历史引用保护
- `⚠️` `staffName` 同时来自表快照和用户昵称补数，改名后口径可能漂移
- `⚠️` 业务员 ID = 用户 ID 是强耦合设计，后续不能轻易拆开
