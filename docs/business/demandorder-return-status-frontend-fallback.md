# 需求单回货状态前端兜底展示

> 更新日期：2026-04-16

## 背景

- 需求单“完结”后，后端会把 `returnStatus` 回写为 `9`，业务文案为 `已完结`。
- 前端需求单列表页原本只通过全局字典 `return_status` 渲染状态文案。
- 如果当前环境的字典缓存还没有刷新到 `9=已完结`，页面虽然已经拿到了 `returnStatus = 9`，但标签文案会显示为空白，看起来像“没有展示出来”。

## 当前处理

- 页面仍然优先使用全局字典 `return_status`。
- 同时在需求单相关列表页增加本地兜底：
  - 当字典里已经有 `9` 时，直接使用字典值。
  - 当字典里还没有 `9` 时，前端补充一个兜底选项：`9 -> 已完结`。

## 影响范围

- `wms-web/src/views/wh/order/demandOrder/index.vue`
- `wms-web/src/views/wh/order/demandOrderContainer/index.vue`

## 维护说明

- 这次兜底是为了解决“字典未刷新导致状态空白”的展示问题，不改变真实业务状态来源。
- 最终状态来源仍然以后端字典和后端返回值为准。
- 如果后续 `return_status` 再扩展新状态，前端展示层也要同步检查：
  - 表格标签文案是否能显示
  - 查询筛选下拉是否能选到
  - 标签颜色映射是否已补齐
