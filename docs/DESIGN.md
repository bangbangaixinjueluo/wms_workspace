# DESIGN

## 本文用途
用于定义设计文档的写作标准、内容深度与评审要求，确保架构设计、模块设计与技术方案保持一致的表达方式。

## 如何维护
- 新增设计文档模板要求时更新本文。
- 设计评审过程发生变化时同步调整。
- 与产品规格或实施计划边界不清时，优先在本文澄清。

## 设计文档应回答的问题
- 为什么需要这个设计。
- 要解决什么问题，目标与非目标是什么。
- 方案如何工作，关键约束和权衡是什么。
- 失败模式、兼容性与回滚方案是什么。

## 推荐结构
- 背景与问题定义
- 目标与非目标
- 方案概述
- 数据结构 / 接口 / 状态变化
- 风险、替代方案与未决问题

## 评审要求
- 关键术语定义清晰。
- 依赖、兼容性、迁移方案不可缺失。
- 所有重要决策需要说明取舍理由。
