# 新用户 Onboarding

## 本文用途
用于描述新用户首次进入系统时的核心流程、关键引导节点、成功标准与后续优化方向。

## 如何维护
- 当注册、初始化配置、首单引导等流程变化时同步更新。
- 需求讨论阶段可先维护在本文，定稿后再拆分更细文档。

## 背景
- 待补充：当前新用户进入系统后的主要痛点。
- 待补充：目标用户画像与使用场景。

## 目标
- 帮助新用户在最短路径内完成首次关键动作。
- 降低首次使用的理解成本与配置门槛。
- 为后续留存与转化建立可观测节点。

## 非目标
- 不覆盖全部高级功能教学。
- 不在首个版本内处理所有个性化配置场景。

## 核心流程
1. 新用户进入系统。
2. 完成必要的身份或组织初始化。
3. 看到首个推荐动作与引导说明。
4. 完成一次关键成功行为。
5. 获得结果反馈与下一步建议。

## 验收标准
- 待补充：关键转化指标。
- 待补充：失败或中断场景的处理方式。
