# 系统首次使用与联调准备

## 目的
本文不是面向 C 端新用户，而是面向第一次接触本项目的开发或实施同学，帮助其在最短路径内跑通一次有效登录和基础联调。

## 适用对象
- 新接手项目的前端开发
- 新接手项目的后端开发
- 实施、测试、联调人员

## 目标
- 能启动前端和后端
- 能完成一次登录
- 能看到动态菜单
- 能验证基础主数据已正常加载

## 最短路径
1. 阅读根目录 [README.md](G:\wms_workspace\README.md)
2. 阅读 [DEPLOYMENT.md](G:\wms_workspace\docs\DEPLOYMENT.md)
3. 启动后端 `yudao-server`
4. 调整前端接口地址或代理
5. 启动前端并登录
6. 检查用户信息、菜单、字典和仓库基础数据

## 首次联调必须确认的内容
- 当前连接的是哪个 profile
- 数据库和 Redis 是否可达
- 前端请求是否真的打到了目标后端
- 登录接口是否成功返回 token
- `GetInfo` 是否返回角色、权限和用户信息
- 是否能正常生成菜单并进入首页

## 常见问题
### 前端能打开但页面空白或无菜单
- 先检查登录后 `GetInfo` 与菜单权限接口是否成功
- 再检查前端是否连到了正确后端

### 后端启动成功但前端仍报接口错误
- 优先检查 `VUE_APP_BASE_API` 和 `vue.config.js` 中的代理目标
- 确认是否仍指向默认远程地址

### 登录后频繁跳回登录页
- 重点检查 token、refresh token、401 刷新逻辑和后端认证配置

## 验收标准
- 能正常登录
- 能访问至少一个 WMS 页面
- 至少一个仓储主数据下拉或列表能正常展示
- 接口文档页可访问
