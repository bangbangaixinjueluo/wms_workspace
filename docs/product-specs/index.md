# 产品规格索引

## 本文用途
用于维护产品规格文档目录，并约定产品需求文档的组织方式、变更记录与生命周期状态。

## 当前目录索引
| 文档 | 状态 | 说明 |
| --- | --- | --- |
| [new-user-onboarding.md](./new-user-onboarding.md) | draft | 新用户 onboarding 流程规格说明。 |
| 后续新增文档 | todo | 新规格文档加入后请同步登记。 |

## 如何维护
- 每个产品规格应对应清晰的用户问题或业务目标。
- 新增文档时同步更新索引与状态。
- 重要变更应在文档内记录版本或变更摘要。

## 生命周期状态建议
- `draft`：草稿，目标与方案仍在完善。
- `aligned`：主要干系人已达成一致。
- `in-build`：已进入开发实现。
- `shipped`：已上线或已交付。
- `retired`：不再维护，仅作历史记录。

## 变更记录建议
- 记录时间、变更人、变更摘要。
- 对范围扩大或缩小的调整要明确说明。
- 对验收标准的修改要保留历史痕迹。
