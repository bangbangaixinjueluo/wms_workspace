# SECURITY

## 当前安全机制
### 认证与授权
- 前端通过 Bearer Token 调用后端接口。
- 前端路由访问基于登录态和权限动态生成。
- 后端保留 Spring Security、菜单权限、按钮权限、租户配置等基础能力。

### 接口安全
- 前端请求层统一注入 token。
- 后端配置中明确了 `permit-all` 白名单接口。
- 验证码能力默认启用。

### 审计基础
- 项目具备操作日志、登录日志、访问日志等框架能力基础。
- 前端还会把页面路径、标题等上下文信息带到请求头，方便审计定位。

## 当前主要风险
### 明文敏感配置
- `wms-api` 的多个环境配置文件中可见数据库、Redis、微信、第三方系统等凭据。
- 这是当前仓库最明显、优先级最高的安全风险。

### 本地配置与真实资源耦合
- 本地 profile 中直接连接实际外部资源，会放大误操作风险。
- 建议将开发资源与真实业务资源彻底隔离。

### XSS 防护关闭
- `application-local.yaml` 中可见 `yudao.xss.enable: false`。
- 这可能是本地调试需要，但需要确认其他环境是否同样关闭。

### Mock/调试开关需确认环境边界
- 本地配置中存在 `security.mock-enable: true`。
- 若没有明确环境隔离和发布检查，容易把调试设置带入不该出现的环境。

## 安全治理建议
### 配置与密钥
- 所有数据库、缓存、第三方 App Secret、OAuth Secret 改为环境变量或密文配置。
- 清理仓库内已有明文，并同步轮换已暴露凭据。

### 权限与接口
- 针对 WMS 关键接口补一份“接口权限矩阵”，明确菜单权限、按钮权限、后端权限码的对应关系。
- 重点复核库存、财务、审批、同步接口是否存在越权修改风险。

### 输入与输出
- 涉及富文本、备注、外部同步数据的接口，需要确认输入校验和输出转义策略。
- 导入、导出、文件上传相关能力建议增加扩展名、MIME 与大小限制说明。

### 发布检查
- 增加发布前安全检查清单：
  - 是否仍使用默认调试开关
  - 是否引用了测试地址
  - 是否保留明文密钥
  - 是否误开文档与调试入口

## 当前优先级最高的动作
1. 迁移并轮换仓库中出现的明文凭据。
2. 核对各 profile 的 `mock`、`xss`、Swagger、管理端点暴露策略。
3. 梳理 WMS 关键写接口的权限码与审计字段。
