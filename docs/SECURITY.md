# SECURITY

## 本文用途
用于记录项目的安全基线与工程要求，覆盖认证授权、输入校验、敏感信息管理、审计与依赖安全。

## 如何维护
- 新增外部集成、权限模型或敏感数据处理时更新。
- 发现安全事件或高风险漏洞后优先沉淀改进规则。

## 安全基线
- 默认拒绝不必要的权限。
- 所有外部输入都视为不可信。
- 敏感信息不得硬编码在仓库中。

## 关键控制点
- 认证与会话管理。
- 授权校验与越权防护。
- 输入校验、输出编码与注入防护。
- 审计日志与关键操作追踪。

## 依赖与配置
- 第三方依赖需要定期审查版本风险。
- 环境变量命名与用途需可追踪。
- 生产配置变更应具备审批与留痕。
