# RELIABILITY

## 本文用途
用于定义系统可靠性建设原则，包括错误处理、重试策略、幂等性、降级、可恢复性与运行期观测要求。

## 如何维护
- 线上事故复盘后优先更新本文。
- 出现新的系统性风险模式时补充相应约定。

## 核心原则
- 关键链路必须能被观测。
- 可重试操作需明确幂等边界。
- 异常不可静默吞掉，应留下定位线索。

## 设计关注点
- 超时与重试策略。
- 外部依赖失败时的降级方案。
- 关键业务操作的一致性保障。
- 批处理或异步任务的失败补偿。

## 运行要求
- 关键事件要有结构化日志。
- 明确区分用户错误、系统错误与外部依赖错误。
- 对可恢复故障定义重试与人工介入边界。
